# Лабораторная работа №5
## Тема: "Обмен данными по именованным каналам"

Программа для управления доступом параллельных процессов к файлу по именованным каналам. Состоит из процессов Server и Client.

## Поток Server:
1. Создает бинарный файл. Имя файла и данные о студентах вводятся с консоли.
2. Выводит созданный файл на консоль.
3. Запускает процессы-клиенты Client, которые выполняют доступ к файлу по именованному каналу. Количество процессов клиентов вводится с консоли.
4. Обслуживает запросы процессов-клиентов.
5. После завершения работы всех процессов клиентов выводит на консоль модифицированный файл.
6. По команде с консоли завершает свою работу.

## Поток Client:
Выполняет цикл: Запрашивает с консоли выполнение одной из операций:
- модификация записи файла;
- чтение записи;
- выход из цикла;

После ввода требуемых данных выполняет эту операцию.

Доступ к записям файла осуществляется по ключу, которым является ID сотрудника.

### При модификации записи поток Client выполняет следующие действия:
1. Запрашивает ключ записи;
2. Посылает запрос на сервер;
3. Выводит полученную с сервера запись на консоль;
4. Запрашивает новые значения полей;
5. По команде с консоли посылает измененную запись на сервер;
6. По команде с консоли завершает доступ к записи.

### При чтении записи поток Client выполняет следующие действия:
1. Запрашивает ключ записи;
2. Посылает запрос на сервер;
3. Выводит полученную с сервера запись на консоль;

Проект создан с использованием Windows API на С++ 17, сборка осуществляется с помощью системы CMake.
