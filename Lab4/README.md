# Лабораторная работа №4
**Тема:** "Синхронизация процессов"

Программа для передачи сообщений между процессами через общий файл, которая включает в себя один процесс Receiver и несколько процессов Sender, которые посылают сообщения процессу Receiver.

## Процесс Receiver выполняет:

1. Ввод с консоли имени бинарного файла и количество записей в бинарном файле, который будет использоваться для передачи сообщений.
2. Создание бинарного файла для сообщений (до 20 символов).
3. Ввод с консоли количества процессов Sender.
4. Запуск заданного количества процессов Sender. Передача каждому процессу Sender через командную строку имя файла для сообщений.
5. Ожидание сигнала на готовность к работе от всех процессов Sender.
6. Циклическое выполнение следующих действий по команде с консоли:
   - Читать сообщение из бинарного файла
   - Завершить свою работу

## Процесс Sender выполняет:

1. Открытие файла для передачи сообщений. Имя файла получить из командной строки.
2. Отправление процессу Receiver сигнала на готовность к работе.
3. Циклическое выполнение следующих действий по команде с консоли:
   - Отправить процессу Receiver сообщение
   - Прекратить свою работу
Текст сообщения вводится с консоли (до 20 символов).

Проект создан с использованием Windows API на С++ 17, сборка осуществляется с помощью системы CMake.
